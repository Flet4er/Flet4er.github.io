Vampire Survivors-like

Командный проект с жестким дедлайном. Моя задача — создать стабильное ядро игры, реализовать все возможные геймплейные механики и уложиться в срок для демонстрации.

Ключевой функционал:

  • Разработал с нуля систему анимации и передвижения для спрайтов (Locomotion System) в UE 5.0, где отсутствовала официальная поддержка 2D-анимаций по состояниям. Система поддерживает 8 (для врагов), 16 (для персонажа) направлений движения и смену состояниия (Idle, Attack, Move) для игрока и всех врагов
  • Реализовал все способности персонажа, свойства предметом, весовую логику выпадения опыта.
  • Выполнил роль баланс-дизайнера: подбирал кривые опыта, настраивал сложность, чтобы достичь целевого хардкорного ощущения в стиле Diablo 2
  • Увеличил FPS через оптимизацию спавна, коллизий и рендера. Уменьшил максимальный размер установочного файла. Увеличил производительность в редакторе UE через рефакторинг кода и Blueprints. Впоследствии перенес проект на UE 5.4, исправил критические ошибки и дополнительно оптимизировал его для порта на Android
  • Обеспечил финальный билд: собрал и протестировал рабочую версию для Android

IsometricChess

Изометрические "шахматы" c большими изменениями в правилах.
Кратко правила: ход игрока состоит из 2 частей: первый - перемещение фигуры, второй - игрок меняет любой тайл в радиусе 2 клетов вокруг своих фигур. Вот некторые тайлы и их эфекты: скала - блокирует перемещение, вода - отнимает -1 к радиусу перемещения, трава - имунитет к изменению тайла. Так же на поле находятся 3 монолита. Для победы необходимо удерживать монолиты, которые приносят очки влияния.

Ключевой функционал:

  • Многоэтапная пошаговая система со сменой ходов и гибкой архитектурой для добавления новых правил
  • Процедурная система изменения игрового поля
  • Интеграция нейросетевого AI с системой поощрения/наказания на основе качества ходов
  • Кастомный UMG Image класс с OnClick-событем для интерактивного взаимодействия
  • Архитектура, оптимизированная для легкой интеграции в мультиплеер
  • Расширяемая система классов (тайлы, фигуры) — новые юниты добавляются без изменения базового кода

MintblomForest

Экспериментальный прототип RPG с модульными системами на Unreal Engine (командный проект)
Разработка велась в коллаборации с техническим художником(https://www.artstation.com/shiiine). Моя зона ответственности — полная реализация игровой логики, архитектура систем и их интеграция.

Ключевой функционал:

  • Модульная система инвентаря по аналогии с Skyrim: данные предметов (стаки, иконки) динамически подгружаются из таблиц DataTable по их уникальному ID
  • Гибкая система крафта: разделена на рецептурную (для экипировки) и комбинаторную (для отваров и еды). Вторая автоматически генерирует новые ID предметов, комбинируя свойства ингредиентов
  • Динамическая система идентификации предметов: префикс "Brew_" для зелий определяет логику расшифровки и применения эффектов
  • Проверка компонентов Nutrition (белки/жиры/углеводы/клетчатка) при создании блюд
  • Гибкая система атрибутов через GameplayTags — добавление любых свойств предметам без изменения кода
  • Система навыков и коллекций - кастомная, на основе Gameplay Ability System (GAS) и Gameplay Tags, что обеспечивает простоту и масштабируемость
  • Оптимизация критичных систем: работа с сохранениями и поиск по DataTable вынесены в C++ для повышения производительности, остальная логика реализована в Blueprint для доработки проекта даже без С++ разработчика

Клиент для игры в DnD

Виртуальный стол для DnD без лишней математики. Мастер загружает карты и фигурки на лету, игроки видят их в реальном времени. Всё, что нужно для живой игры на расстоянии: создание персонажей, броски костей

Ключевой функционал:

  • Проект полностью написан на С++. Blueprint использовался только в UI.
  • HTTP-сервер для рантайм-загрузки а так же репликации и синхронизации ассетов(изображений)
  • Динамическая загрузка карт - изображения загружаются во время игры и реплицируются через HTTP сервер
  • Система сохранения/загрузки партий с восстановлением состояния. Рантайм-изображения сохраняются в исходном виде
  • Мультиплеер с репликацией на 6 игроков. Система хостинга сессий и поиск доступных игр
