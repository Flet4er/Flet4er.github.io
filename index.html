Vampire Survivors-like: от прототипа до оптимизированного мобильного билда

Командный проект с жестким дедлайном. Моя задача — создать стабильное ядро игры, интегрировать работу команды и уложиться в срок для демонстрации. -Разработал с нуля систему анимации и передвижения для спрайтов (Locomotion System) в UE 5.0, где отсутствовала официальная поддержка 2D-анимаций по состояниям. Система поддерживала 16 направлений движения и смену спрайтов (Idle, Attack, Move) для игрока и всех врагов. -Поднял FPS с 30 до 80+ через оптимизацию спавна, коллизий и рендера. Впоследствии перенес проект на UE 5.4, исправил критические ошибки и дополнительно оптимизировал его. -Выполнил роль баланс-дизайнера: подбирал кривые опыта, настраивал сложность, чтобы достичь целевого хардкорного ощущения в стиле Diablo 2. -Обеспечил финальный билд: собрал и протестировал рабочую версию для ПК и Android, выполнив первоначальную задачу проекта. Разработка кастомной системы управления спрайтовыми анимациями (аналог AnimBP) для UE4.27, когда штатный функционал Paper2D был ограничен

MintblomForest

«Экспериментальный прототип RPG с модульными системами на Unreal Engine (командная разработка)» Разработка велась в коллаборации с техническим художником(https://github.com/Shiiine). Моя зона ответственности — полная реализация игровой логики, архитектура систем и их интеграция. Небольшой, но технически насыщенный проект, демонстрирующий умение проектировать и реализовывать, как заимствованные, так и создавать новые сложные игровые системы с гибкой архитектурой. Ключевой функционал: Модульная система инвентаря по аналогии с Skyrim: данные предметов (стаки, иконки) динамически подгружаются из таблиц DataTable по их уникальному ID. Гибкая система крафта: разделена на рецептурную (для экипировки) и комбинаторную (для зелий и еды). Вторая автоматически генерирует новые ID предметов, комбинируя свойства ингредиентов. Система навыков и коллекций на основе Gameplay Ability System (GAS) и Gameplay Tags, что обеспечивает масштабируемость и простоту добавления нового контента. Оптимизация критичных систем: работа с сохранениями и поиск по DataTable вынесены в C++ для повышения производительности, остальная логика реализована в Blueprint для скорости итераций.
